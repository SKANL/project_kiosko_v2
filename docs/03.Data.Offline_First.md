# 03. Data: Offline-First Strategy

> **Status:** Draft
> **Key Tech:** SQLite (`sqlite-net-pcl`), System.Text.Json

## 1. The "Zero-Trust in Network" Principle

Nodus assumes the network **does not exist** by default. All data operations (Reads/Writes) happen against the Local Database first. Network sync is a secondary background process.

## 2. Local Database Schema (SQLite)

We use a normalized schema optimized for read-heavy UI and atomic write-heavy voting.

### Key Tables

#### `Events`

Details about the current hackathon/expo.

```csharp
class Event {
    [PrimaryKey] string Id {get;set;} // UUID
    string Name {get;set;}
    string EvaluationRubricJson {get;set;} // Dynamic JSON definition of criteria
    int RubricVersion {get;set;}           // Incremented each time the rubric changes; used to detect stale judge caches
                                           // IMMUTABLE once the first Vote exists for this event (Decision #46).
    string GlobalSalt {get;set;}           // Per-event salt for crypto operations
    string SharedAesKeyEncrypted {get;set;} // PBKDF2-derived AES key (Decision #22) stored encrypted; used to decrypt judge_register packets (Decision #31)
}
```

#### `Projects`

The "Targets" to be evaluated. Populated via **Live Registration**.

```csharp
class Project {
    [PrimaryKey] string Id {get;set;} // Auto-assigned "PROJ-xyz"
    string EventId {get;set;}         // FK → Events.Id. Required (Decision #30) — a project always belongs to exactly one event.
    int SequenceNumber {get;set;}     // Auto-incremented per event. Used as delta cursor for project_sync: Admin returns WHERE SequenceNumber > since_seq (Decision #39).
    string Name {get;set;}
    string Category {get;set;} // "Software", "Hardware"
    string? GithubUrl {get;set;} // Optional
    string? Members {get;set;} // Optional
    string? Description {get;set;} // Optional. Max 140 chars. Student-submitted "Twitter-style" pitch (Doc 14 §1).
    string? StandNumber {get;set;} // Admin-assigned (e.g., "A-12"). Null until assigned → displays as "Unassigned" (Decision #26).
    string? EditToken {get;set;}   // System-generated UUID. Allows student to reopen registration form to correct typos. Valid until Admin closes event (Doc 14 §1, Doc 15 §2).
}
```

#### `Judges`

Registered judges. Populated during the `judge_register` GATT handshake (Decision #8). Admin uses this table to verify Ed25519 signatures and enforce the blocklist (Decision #28).

```csharp
class Judge {
    [PrimaryKey] string Id {get;set;}   // UUID generated on first registration
    string EventId {get;set;}           // FK → Events.Id
    string Name {get;set;}              // As entered in Nodus.Judge app
    string PublicKeyBase64 {get;set;}   // Ed25519 public key (Base64) — used to verify vote packet signatures
    bool IsBlocked {get;set;} = false;  // If true: all packets signed with this key are silently rejected
    DateTime RegisteredAtUtc {get;set;}
}
```

#### `Votes` (The Critical Transaction)

```csharp
class Vote {
    [PrimaryKey] string Id {get;set;} // GUID generated on creation
    string EventId {get;set;}
    string ProjectId {get;set;}
    string JudgeId {get;set;}
    int Version {get;set;} = 1; // Incremented on re-evaluation. Server applies "Latest Version Wins" by Version (higher wins). CreatedAtUtc kept for audit only (Decision #34).

    string PayloadJson {get;set;} // The full scores { "Design": 10, "Code": 5 }

    // Multimedia References
    string? LocalPhotoPath {get;set;}   // Device-local path; null if no photo
    string? LocalAudioPath {get;set;}   // Device-local path; null if no audio
    string? RemotePhotoUrl {get;set;}   // MongoDB GridFS URL set by Admin after Wi-Fi/Mule upload; null until MediaStatus = ReachedCloud (Decision #47)
    string? RemoteAudioUrl {get;set;}   // MongoDB GridFS URL set by Admin after Wi-Fi/Mule upload; null until MediaStatus = ReachedCloud (Decision #47)

    // Sync State — two independent axes (Decision #24)
    // ScoreStatus: tracks PayloadJson (scores) via BLE Swarm path
    // MediaStatus: tracks photos/audio via Wi-Fi or BLE Mule Mode path
    // RULE (Decision #47): if LocalPhotoPath == null AND LocalAudioPath == null, set MediaStatus = ReachedCloud on creation (nothing to sync).
    SyncStatus ScoreStatus {get;set;} = SyncStatus.Pending;
    SyncStatus MediaStatus {get;set;} = SyncStatus.Pending;
    DateTime CreatedAtUtc {get;set;}
    DateTime? SyncedAtUtc {get;set;}
}

// Pending      → data has not left the device
// ReachedAdmin → data arrived at Admin node (BLE for scores; Wi-Fi or Mule Mode for media)
// ReachedCloud → Admin synced to MongoDB via Nodus.API
// SyncError    → failed after retries
enum SyncStatus { Pending, ReachedAdmin, ReachedCloud, SyncError }
```

## 3. Synchronization Logic

### A. The "Lightweight" Payload (Bluetooth)

We **strictly separate** metric data (numbers/text) from heavy media (photos/audio).
When a Judge presses "Submit":

1.  **Atomic Save:** The Vote is saved to SQLite with `ScoreStatus = Pending, MediaStatus = Pending`.
2.  **BLE Attempt:** The App attempts to send _strictly_ the `PayloadJson` (Scores) via Bluetooth Swarm.
3.  **Result:**
    - _Success:_ Update `ScoreStatus = ReachedAdmin` (scores confirmed at Admin node).
    - _Failure:_ Keep `ScoreStatus = Pending`. Background worker retries later.

### B. The "Heavyweight" Sync (Media Files)

Media files (photos, voice notes) follow a **two-tier strategy**:

**Primary: Wi-Fi upload (preferred)**
1.  Files reside on the device's internal storage.
2.  When the Judge connects to Wi-Fi, a `MediaSyncJob` triggers automatically.
3.  Files are uploaded to the **Nodus API** (`POST /api/media/upload`), which stores them in **MongoDB GridFS** and returns the remote URL.
4.  Admin updates `Vote.RemotePhotoUrl` and/or `Vote.RemoteAudioUrl` with the returned URLs and advances `MediaStatus = ReachedCloud` (Decision #47).

**Fallback: BLE in Mule Mode (no Wi-Fi available)**
1.  If the Judge is in **Mule Mode** (no BLE signal from Server or any LINK node for > 10 min, Decision #23) **and** physically approaches the Admin laptop (RSSI > -60 dBm), a BLE `MediaSyncService` activates.
2.  Files are chunked and transferred directly to the Admin node via GATT Write (see Doc 10).
3.  The Admin node stores them locally and syncs to MongoDB when internet becomes available.

> **Decision:** Wi-Fi is always the primary media transport. BLE media transfer is an emergency fallback and only activates in Mule Mode proximity.

## 4. Conflict Resolution & Reliability

### "Append-Only" Philosophy

Votes are **never edited in-place**. If a Judge re-evaluates a team (via the "Re-evaluate" screen):

1.  A **new** Vote record is created with `version` incremented (e.g., `version: 2`).
2.  The UI pre-loads previous scores for convenience, but a new database row is always inserted.
3.  Server Logic applies **"Latest Version Wins"** based on the `version` field (higher `version` wins). `CreatedAtUtc` is kept for audit only (Decision #34).
4.  This avoids the nightmare of merging conflict states on 50 distributed devices.

### Transaction Integrity

All SQLite writes use `RunInTransaction` to ensure that a Vote isn't partially saved (e.g., Header saved but Scores missing) if the battery dies mid-write.
