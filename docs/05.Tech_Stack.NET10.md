## 3. Architecture Pattern

**MVVM (Model-View-ViewModel)** with **Clean Architecture** services.

- `Services/`: `BluetoothService`, `SyncService`, `CryptoService`.
- `ViewModels/`: `VotingViewModel`, `HomeViewModel`.
- `Models/`: Each app owns its own models. There is **no shared library** (`Nodus.Shared` does not exist). Models are duplicated across apps intentionally to keep each app independently deployable.

## 3.2 Nodus.Web Deployment Inside Nodus.Admin (Decision #52)

`Nodus.Web` is a **Blazor WebAssembly Standalone** app — it compiles to a set of static files (`index.html`, `*.wasm`, `*.js`, `*.css`). These files are **embedded in and served by Nodus.Admin's built-in Kestrel HTTP server**. No separate hosting process or internet connection is required.

### How It Works

1. **Build step:** `dotnet publish Nodus.Web` produces a `wwwroot/` folder of static files.
2. **Embedding:** The Nodus.Admin `.csproj` references the published output folder as embedded resources **or** copies them to a known subdirectory (e.g., `{AppData}\Nodus\Web\`) during installation.
3. **Kestrel:** On startup, `Nodus.Admin` launches an embedded Kestrel HTTP server on `0.0.0.0:5000` and maps the static files root to the embedded `wwwroot/` directory.
4. **Endpoint exposure:**
   - `GET /` → serves `index.html` (Blazor bootstrap)
   - `GET /register?event=EVT-ID` → Blazor routing handles this
   - `GET /edit?token=UUID` → Blazor routing handles this
   - `GET /api/event/current` → Kestrel minimal API endpoint (Decision #19)
   - `POST /api/projects` → Kestrel minimal API endpoint (project registration)
5. **Network:** Only accessible on the local Wi-Fi network. Students open `http://[LOCAL_IP]:5000` from their phone browsers.

### Implementation (.NET 10)

```csharp
// In Nodus.Admin startup (e.g., MauiProgram.cs or App.xaml.cs)
var builder = WebApplication.CreateBuilder();
builder.WebHost.UseUrls("http://0.0.0.0:5000");

var app = builder.Build();
app.UseDefaultFiles();
app.UseStaticFiles(new StaticFileOptions {
    FileProvider = new PhysicalFileProvider(
        Path.Combine(AppContext.BaseDirectory, "Web")  // published Blazor WASM output
    )
});

// Minimal API routes
app.MapGet("/api/event/current", () => eventService.GetCurrentEvent());
app.MapPost("/api/projects", async (ProjectDto dto) => await projectService.RegisterAsync(dto));

_ = app.RunAsync(); // fire-and-forget; does not block the MAUI UI
```

> **Windows Firewall:** The installer (or first-run setup wizard) must add a Windows Firewall inbound rule for port 5000 TCP. Without it, students on the same Wi-Fi cannot reach the registration page.

## 4. Development Environment

- **IDE:** Visual Studio 2026 (Preview) or VS Code with C# Dev Kit.
- **Hardware:**
  - Must have a physical Android Device (Emulator does NOT support Bluetooth).
  - Must have a Windows Laptop with Bluetooth 5.0+ support (or Dongle).

## 5. Critical Code Convention

**Async/Await:** Strict usage. BLE operations are inherently asynchronous.
**CancellationTokens:** All BLE operations must support cancellation to prevent "hanging" connection attempts.
