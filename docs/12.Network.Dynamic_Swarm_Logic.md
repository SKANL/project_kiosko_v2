# 12. Network: Dynamic Swarm Logic (The "Firefly" Protocol)

> **Status:** Active
> **Concept:** Ephemeral, rotating chains based on Signal Quality (RSSI) + Randomization.

## 1. The Core Concept

Instead of static "Relays", every node acts like a firefly. It shines (Advertises) for a short burst if it's in a good position, then goes dark (Scan only) to save battery and let others take the load.

## 2. The Factors (Inputs)

Every **1-5 seconds** (canonical interval per [Doc 02](02.Network.Swarm_Protocol.md) — variable to prevent node synchronization), a Node evaluates its `SuitabilityScore`:

1.  **Server RSSI:** "Can I see the target?" (Primary Factor)
2.  **Battery:** "Can I afford to shout?" (Veto Factor < 20%)
3.  **Random Noise:** "Am I the chosen one?" (Collision Avoidance)

## 3. The State Machine (FSM)

> **⚠️ Canonical Source:** The Firefly FSM — states, transitions, parameters, and state names — is defined exclusively in **[Doc 02: Network — Swarm Protocol](02.Network.Swarm_Protocol.md)** (Decision #3). This document focuses only on the suitability scoring logic that feeds into the FSM transitions.
>
> States for reference: `SEEKER` → `CANDIDATE` → `LINK` → `COOLDOWN`. Do not redefine them here.

## 4. Addressing the Risks

### A. Battery Drain

**Mitigation:** The "Rotation" ensures no single device dies. If 50 devices are present, each might only be a Link for 2 minutes per hour.

### B. iOS Limitation (The "Elephant in the Room")

**Rule:** _This logic ONLY runs when the App is FOREGROUND._

- If the user locks the phone:
  - **Android:** Can continue (with Sticky Notification).
  - **iOS:** IMMEDIATELY sends "Goodbye" packet and stops Advertising. Logic aborts.

void OnHeartbeat() {
    if (Mode == LINK && TimeActive > 60s) {
        BecomeCooldown();
        return;
    }

    if (Mode == SEEKER && ServerRssi > -75) { // Canonical threshold: -75 dBm (see Doc 02)
        // Don't switch instantly (thundering herd problem)
        // Wait canonical random interval: 3s-15s (see Doc 02)
        ScheduleCandidateCheck();
    }
}

double CalculateLinkScore() {
    // Battery veto: GetBatteryLevel() returns 0.0-1.0 scale (Decision #53).
    // Threshold is 0.20 (not 20). Using < 20 would ALWAYS veto since values are <= 1.0.
    if (bleManager.GetBatteryLevel() < 0.20) return 0; // battery veto

    return ComputeRssiScore() + ComputeRandomNoise();
}
