## 2. Unit Testing the Algorithm (xUnit)

We don't test the Bluetooth hardware; we test the **Logic**.

### Mocking the `IBleManager`

```csharp
public interface IBleManager {
    int GetServerRssi();
    double GetBatteryLevel(); // Returns 0.0–1.0 (e.g. 0.15 = 15%). Canonical scale: 0.0–1.0.
}

// The Test
[Fact]
public void Candidate_Should_NOT_Become_LINK_If_Battery_Low() {
    var mockBle = new Mock<IBleManager>();
    mockBle.Setup(x => x.GetBatteryLevel()).Returns(0.15); // 15% — below the 20% veto threshold (Decision #3)

    var swarm = new SwarmService(mockBle.Object);
    var score = swarm.CalculateLinkScore();

    Assert.Equal(0, score); // Must be zero: battery veto < 0.20 forces score = 0
}
```

## 3. Integration Testing (The "Virtual Lab")

We create a console app `Nodus.Simulator.exe` that instantiates 100 `VirtualClient` objects in memory.

### The Simulation Loop

1.  **Setup:** Create 1 Server, 100 Clients.
2.  **Topology:** Assign random X,Y coordinates to each client.
3.  **Physics:** Calculate `RSSI` based on `Distance(Server, Client)`.
4.  **Run:**
    - Tick 1: All Clients scan. FSM loop begins.
    - Tick 2: **Trickle Check:** Candidates count active neighbors (`k`).
    - Tick 3: **Suppression:** Verify that if `Neighbors >= 2`, Candidate -> SEEKER (Does NOT become LINK).
    - Tick 4: **Election:** Remaining Candidates become **LINK** nodes.
    - Tick 5: Verify that SEEKER clients find a new LINK node.

## 5. Blocklist & Event-Close Test Cases

These scenarios must be covered in the xUnit suite:

### A. `blocklist_sync` Enforcement

```csharp
[Fact]
public void Link_Node_Should_Drop_Packets_From_Blocked_Key() {
    var blockedKey = "BASE64_ED25519_KEY";
    var relay = new RelayService();
    relay.ApplyBlocklist(new[] { blockedKey });

    var packet = BuildPacket(signedWith: blockedKey);
    var result = relay.ShouldRelay(packet);

    Assert.False(result); // LINK node must drop packets from blocked keys
}

[Fact]
public void Admin_Should_Reject_Packets_From_Blocked_Key_Even_If_Relay_Passed_It() {
    var blockedKey = "BASE64_ED25519_KEY";
    var admin = new AdminService();
    admin.BlockJudge(blockedKey);

    var packet = BuildPacket(signedWith: blockedKey);
    var result = admin.TryAcceptVote(packet);

    Assert.Equal(VoteAcceptResult.Rejected_BlockedKey, result);
}
```

### B. `event_status: "closed"` Flow

```csharp
[Fact]
public void Judge_Should_Stop_Polling_When_EventStatus_Is_Closed() {

    Assert.False(judge.IsPollingActive);
```

## 4. Load Testing: The "Judge Storm" Scenario

Verify that Nodus.Admin can handle concurrent GATT connections from many judges registering and syncing simultaneously.

### Test Objectives

1. **`project_sync` thundering herd:** Even with `Random(50s, 70s)` jitter (Decision #43), measure how many simultaneous `NODUS_BOOTSTRAP_READ` reads Nodus.Admin can serve before queueing or dropping connections. Target: sustain **15 concurrent** GATT reads without data loss.
2. **Vote ingestion rate:** Measure Admin throughput when 50 judges submit votes close together at the end of a session. Target: process **100 votes within 30 seconds** without `SyncError`.
3. **Memory footprint:** Monitor Admin SQLite write latency under concurrent vote writes. Transactions must stay atomic (Doc 03 §4). Target: no partial saves detected.

### Implementation

Extend `Nodus.Simulator.exe` (Section 3) with a stress profile:

```csharp
// Spawn N VirtualJudge instances that all poll BOOTSTRAP_READ within a 20s window
for (int i = 0; i < 50; i++) {
    var judge = new VirtualJudge(id: i, pollJitter: Random.Shared.Next(0, 20_000));
    judge.Start();
}
// Assert: Admin reports zero SyncError in the VirtualAdmin state after 60s
Assert.Equal(0, VirtualAdmin.SyncErrorCount);
Assert.Equal(50, VirtualAdmin.VotesReceived);
```

### Battery Scale Reminder (Decision #53)

All battery comparisons use the **`0.0–1.0` scale**. The battery veto threshold is `< 0.20` in code — NOT `< 20`. The mock returning `0.15` represents 15% battery.

```csharp
// CORRECT
mockBle.Setup(x => x.GetBatteryLevel()).Returns(0.15); // 15% — triggers veto (0.15 < 0.20)
Assert.Equal(0, swarm.CalculateLinkScore());

// WRONG — do not use integer percentages
// mockBle.Setup(x => x.GetBatteryLevel()).Returns(15); // BUG: 15 > 0.20 → no veto fires
```
